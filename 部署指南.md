# AERONEX CI/PL Generator - 云端部署指南

## 目标
部署到 Render.com（免费），打开链接就能用，不需要本地启动任何东西。

---

## 部署步骤（5分钟）

### 第一步：注册 Render.com
1. 打开 https://render.com
2. 点击 "Get Started for Free"
3. 选择 "Continue with GitHub"（用GitHub账号登录）

### 第二步：把代码推到 GitHub
**如果你已经有GitHub账号：**

1. 在 GitHub 上创建一个新仓库（Repository），名字比如 `aeronex-cipl`
2. 把本项目的所有文件推到这个仓库

```bash
# 在项目根目录执行
cd /mnt/agents/output/app
git init
git add .
git commit -m "Initial deployment"
git branch -M main
git remote add origin https://github.com/你的用户名/aeronex-cipl.git
git push -u origin main
```

### 第三步：在 Render 上部署
1. 登录 Render 后，点击 "New +" 按钮
2. 选择 "Blueprint"
3. 连接你的 GitHub 账号，选择 `aeronex-cipl` 仓库
4. Render 会自动识别 `render.yaml` 文件
5. 点击 "Apply" 开始部署
6. 等待 3-5 分钟（免费计划需要等待构建完成）

### 第四步：拿到你的链接
部署完成后，Render 会给你的服务一个公网URL：
```
https://aeronex-cipl-generator.onrender.com
```

打开这个链接，上传PDF，解析完全在云端完成。

### 第五步：分享给其他人
直接把这个 URL 发给同事/朋友，他们打开就能用，不需要安装任何东西。

## 常见问题

**Q: 免费计划有什么限制？**
- 15分钟无访问会自动休眠，下次访问需要等10秒唤醒
- 每月750小时免费额度（够一个人24小时用31天）
- 如果需要一直在线，可以设置定时ping（后面会教）

**Q: 构建失败了怎么办？**
- 检查 Render 的 Logs 页面，看错误信息
- 通常是缺少依赖，检查 `python_backend/requirements.txt` 是否完整
- 或者 Dockerfile 中的命令有误

**Q: 如何更新代码后重新部署？**
- 修改代码 → `git push` → Render 自动重新部署

## 如果不想用GitHub

Render 也支持直接从代码部署：
1. 点击 "New +" → "Web Service"
2. 选择 "Deploy an existing image from a registry"
3. 或者选择 "Upload your code"
4. Runtime 选 `Docker`
5. 免费计划，点击 Create
